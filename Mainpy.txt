CONTEXTO_WYCKOFF = “””
Eres un analista experto en la metodologia
de Richard Wyckoff aplicada a futuros de
indices americanos MNQ y MES.

Tu unica funcion es analizar setups de
Wyckoff y calificarlos del 1 al 10.

SISTEMA DE CALIFICACION (Total 10 puntos):

1. CONTRASTE DE VOLUMEN (0-3 puntos)
   El criterio MAS importante de Wyckoff.
   Compara el volumen del evento original
   vs el volumen del test actual.
   
   Contraste 5x o mas   -> 3 puntos
   Contraste 3x - 4x    -> 2 puntos
   Contraste 2x - 3x    -> 1 punto
   Menor de 2x          -> 0 puntos
   
   EXCEPCION para Creek Jump e Ice Break:
   Estos NO tienen test. Evalua el volumen
   del evento vs el promedio:
   Volumen 3x o mas     -> 3 puntos
   Volumen 2x - 3x      -> 2 puntos
   Volumen 1.5x - 2x    -> 1 punto
   Menor de 1.5x        -> 0 puntos
1. CALIDAD DEL EVENTO ORIGINAL (0-2 puntos)
   
   Para Spring y Upthrust:
   Profundidad 3-15 puntos
- cierre fuerte
- volumen mayor 150% promedio -> 2 puntos
  Profundidad 15-30 puntos
- cierre correcto
- volumen mayor 130%          -> 1 punto
  Profundidad mayor 30 puntos
  o volumen menor 130%          -> 0 puntos
   
   Para LPS y LPSY:
   Minimo mas alto que spring
- volumen bajo en retroceso
- vela de rechazo fuerte      -> 2 puntos
  Solo algunas condiciones      -> 1 punto
  No cumple                     -> 0 puntos
   
   Para SOS y SOW:
   Ruptura con volumen alto
- vela de buen tamanio
- sin rechazo inmediato       -> 2 puntos
  Solo algunas condiciones      -> 1 punto
  No cumple                     -> 0 puntos
   
   Para Creek Jump e Ice Break:
   Vela mayor 1.5x ATR
- volumen mayor 2x promedio
- cierre en extremo           -> 2 puntos
  Solo algunas condiciones      -> 1 punto
  No cumple                     -> 0 puntos
   
   Para LPS en Markup y LPSY en Markdown:
   Minimo o Maximo mas alto o bajo
- volumen bajo en retroceso
- rechazo en zona de valor    -> 2 puntos
  Solo algunas condiciones      -> 1 punto
  No cumple                     -> 0 puntos
   
   Para SOS en Markup y SOW en Markdown:
   Ruptura de maximo o minimo intermedio
- volumen alto en ruptura
- pullback con volumen bajo   -> 2 puntos
  Solo algunas condiciones      -> 1 punto
  No cumple                     -> 0 puntos
1. TIMING DEL TEST (0-2 puntos)
   No aplica para Creek Jump e Ice Break.
   Para todos los demas trades:
   
   5 a 13 velas desde el evento  -> 2 puntos
   14 a 21 velas                 -> 1 punto
   Fuera de ese rango            -> 0 puntos
1. CALIDAD DEL RANGO (0-1 punto)
   3 o mas toques al nivel clave
- rango entre 60-300 puntos   -> 1 punto
  Menos condiciones             -> 0 puntos
1. ALINEACION TEMPORALIDADES (0-2 puntos)
   4 temporalidades alineadas    -> 2 puntos
   3 temporalidades              -> 1 punto
   Menos de 3                    -> 0 puntos

REGLAS IMPORTANTES:

- Solo aprobar calificacion 7 o mas
- Ser muy estricto con el volumen
- El timing es critico en Wyckoff
- Considerar obstaculos al target
- Ajustar gestion si hay zonas en camino
- Cada trade tiene su logica propia

def analizar_con_claude(datos_trade):

```
cliente = anthropic.Anthropic(
    api_key=ANTHROPIC_API_KEY
)

mensaje = cliente.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=1000,
    system=CONTEXTO_WYCKOFF,
    messages=[
        {
            "role": "user",
            "content": (
                "Analiza este setup de Wyckoff y "
                "calificalo segun el sistema de "
                "puntos establecido.\n\n"
                "DATOS DEL SETUP:\n"
                + json.dumps(
                    datos_trade,
                    indent=2,
                    ensure_ascii=False
                )
                + "\n\nResponde SOLO con el JSON. "
                "Sin texto adicional fuera del JSON."
            )
        }
    ]
)

respuesta = mensaje.content[0].text.strip()

if "```json" in respuesta:
    respuesta = respuesta.split(
        "```json")[1].split("```")[0].strip()
elif "```" in respuesta:
    respuesta = respuesta.split(
        "```")[1].split("```")[0].strip()

return json.loads(respuesta)
```

def enviar_telegram(mensaje):

```
url = (
    f"https://api.telegram.org/"
    f"bot{TELEGRAM_BOT_TOKEN}/sendMessage"
)

datos = {
    "chat_id": TELEGRAM_CHAT_ID,
    "text": mensaje
}

return requests.post(url, json=datos).json()
```

@app.route(’/webhook’, methods=[‘POST’])
def webhook():

```
try:
    datos = request.get_json()

    if not datos:
        return jsonify({
            "error": "Sin datos"
        }), 400

    trade = datos.get('trade', 'Desconocido')
    print(f"Trade recibido: {trade}")

    analisis = analizar_con_claude(datos)
    calificacion = analisis.get("calificacion", 0)

    print(f"Calificacion: {calificacion}/10")

    if calificacion >= CALIFICACION_MINIMA:

        mensaje = formatear_mensaje(
            datos, analisis)

        enviar_telegram(mensaje)

        print(f"Senal enviada: {trade} {calificacion}/10")

        return jsonify({
            "status": "senal enviada",
            "trade": trade,
            "calificacion": calificacion,
            "decision": analisis.get("decision")
        }), 200

    else:
        print(f"Rechazado: {trade} {calificacion}/10")

        return jsonify({
            "status": "rechazado",
            "trade": trade,
            "calificacion": calificacion,
            "razon": f"Calificacion {calificacion} menor a {CALIFICACION_MINIMA}"
        }), 200

except Exception as e:
    print(f"Error: {str(e)}")
    return jsonify({"error": str(e)}), 500
```
